# Contributing to SxnnyUI

Thank you for your interest in contributing to SxnnyUI — contributions of all kinds are welcome.

Please follow these guidelines to make the review process smooth and fast:

1. Open an issue first for non-trivial changes or features so we can discuss scope and design.
2. Fork the repository and create a feature branch from `main`:

```bash
git clone git@github.com:Sxnnyside-Project/SxnnyUI.git
git checkout -b feat/your-feature
```

3. Write tests for new features or bug fixes where applicable. Run the test suite with:

```bash
swift test
```

4. Keep commits focused and atomic. Use present-tense, imperative commit messages (e.g. `docs: add contributing guide`).

5. Update documentation and `README.md` when you add or change public APIs.

6. Submit a pull request against `main`. Fill out the PR template and link any related issues.

Review expectations
- PRs are reviewed for correctness, clarity, and API stability.
- Be responsive to reviewer feedback and keep PRs small when possible.
- Major API changes should be discussed in an issue before implementation.

Code style
- Follow the existing code style in the repository. Prefer clarity over cleverness.
- If the project includes a formatter configuration, run it before committing.

Security
- If you discover a security issue, see `SECURITY.md` for private reporting instructions.

Thanks again — your contributions help make SxnnyUI better for everyone.
